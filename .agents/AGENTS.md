# Authority Project Harness Rules

## 하네스: Authority 풀스택 팀
**목표:** Supabase 데이터베이스와 React/Vite 기반 프론트엔드를 활용하여 빠르고 안정적인 웹 애플리케이션 개발 워크플로우를 제공합니다.

**팀 프로토콜:**
1. **오케스트레이터(authority-orchestrator):** 기능 개발 지시가 떨어지면 전체 팀을 지휘합니다.
2. **DB Admin(db-admin):** Supabase MCP를 활용하여 DB 스키마 검증 및 수정을 담당합니다.
3. **Frontend Dev(frontend-dev):** React/Vite로 UI를 구현하며, Supabase 클라이언트를 연동합니다.
4. **QA/Reviewer(qa-reviewer):** 코드 리뷰, 테스트, GitHub PR/Push 등 배포 전 단계를 담당합니다.

**실행 규칙:**
- 각 에이전트는 담당 스킬 파일의 가이드라인에 따라 행동합니다.
- 변경 사항은 필요에 따라 GitHub(Authority 저장소)에 커밋하고 푸시해야 합니다.
- 개발 진행 중에는 에이전트 간의 결과물 교환을 위해 명확한 커뮤니케이션과 파일 검증을 진행합니다.
